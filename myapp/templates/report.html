Отчет по разработке сайта на Django

Подготовка

Сначала на рабочем столе создаётся папка проекта с именем «Myproject», после чего она открывается в среде разработки PyCharm Community Edition. Затем необходимо убедиться, что установлен актуальная версия Python. Для этого в консоли(Рисунок 1) вводится команда:

 python --version

Рисунок 1 — Проверка версии Python.


В случае, если Python не установлен необходимо установить его с официального сайта.

Установка Django

После успешной установки Python необходимо добавить фреймворк Django. Для этого в командной строке выполняется следующая команда:

 pip install django

Этот шаг обеспечивает установку последней версии Django для дальнейшей работы над проектом.(Рисунок 2)

Рисунок 2 — Установка Django.

Создание проекта

Следующим шагом создаётся новый Django-проект с помощью команды:

 django-admin startproject [name]

где [name] — это название вашего проекта (например, Myproject).

После создания проекта нужно перейти в его директорию командой:

 cd [name]

Эти действия позволяют начать работу с проектом и выполняются, как показано на Рисунке 3.

Рисунок 3 — Создание и открытие проекта.

Создание приложения

В Django приложение (app) предназначено для логической организации функционала проекта — оно группирует связанные компоненты, такие как модели, представления и шаблоны, обеспечивая четкую структуру кода. Чтобы создать новое приложение, выполните в командной строке:

 python manage.py startapp [name]

где [name] — название приложения (например, main, blog и т. д.).

Этот шаг показан на Рисунке 4. После выполнения команды в проекте появится папка с базовой структурой приложения, готовая для дальнейшей разработки.

Выполнение миграций

Следующий этап — применение миграций (Рисунок 8). Для этого используются команды:

            python manage.py makemigrations
            python manage.py migrate

Без миграций разработчикам пришлось бы вручную создавать SQL-скрипты для каждой правки в моделях. Это увеличивает риск ошибок, усложняет процесс разработки и замедляет работу над проектом. Миграции же обеспечивают контроль версий для БД, упрощая командную работу и откат изменений при необходимости.

Создание форм

Следующим этапом является создание форм для регистрации и авторизации, которые необходимы в соответствии с требованиями к разработке данного проекта. Чтобы создать формы регистрации и авторизации необходимо создать файл по пути «myproject/myapp» с названием «forms.py» как показано на Рисунке 9.

Рисунок 9 — Создание форм.


На рисунке 9 представлены две формы: «CustomUserCreationForm» и «CustomAuthenticationForm». Первая форма, «CustomUserCreationForm», предназначена для регистрации новых пользователей. Она позволяет заполнить поля «Имя пользователя», «Электронная почта», «Пароль1» и «Пароль2», которые затем используются для создания новой записи в таблице «Пользователи». Создание представлений

Представления играют ключевую роль в функциональности приложения, обеспечивая взаимодействие с пользователем, обработку его действий и отображение необходимой информации. Кроме того, представления отвечают за взаимодействие с базой данных (БД). Как показано на рисунке 10, файлы представлений (views.py) обычно размещаются в каталоге приложения: myproject/myapp/views.py. Именно в этих файлах определяется логика обработки запросов пользователей, извлечения данных из БД, их обработки и, наконец, формирования ответа, который будет показан пользователю. Например, здесь может быть логика регистрации новых пользователей, авторизации, отображения списка товаров или редактирования профиля.

Настройка URL

Перед запуском разрабатываемого проекта необходимо настроить URL-адреса. Каждый URL-адрес, который вводит пользователь в браузере, должен быть сопоставлен с конкретным представлением (функцией или классом). Это необходимо для того, чтобы Django знал, какой код нужно выполнить для обработки запроса пользователя и возврата соответствующего ответа. Например, URL-адрес /register/ может быть связан с представлением, обрабатывающим регистрацию пользователя, а URL-адрес /login/ — с представлением для авторизации. Правильная настройка URL-адресов критически важна для навигации по сайту и функционирования приложения в целом. Неправильно настроенные URL-адреса могут приводить к ошибкам и неработоспособности отдельных разделов сайта.


Настройка URL

Последним шагом перед демонстрацией работоспособности приложения является запуск локального веб-сервера. Для этого используется команда, которая запускает встроенный в Django сервер разработки: 

 python manage.py runserver

После успешного запуска сервера приложение будет доступно в веб-браузере по адресу http://127.0.0.1:8000/. Этот адрес указывает на локальный компьютер (127.0.0.1) и порт 8000, на котором работает сервер. Открыв этот адрес в браузере, можно увидеть главную страницу приложения и проверить его работоспособность. Любые изменения в коде приложения будут автоматически загружаться сервером, что упрощает процесс разработки и отладки. Важно отметить, что этот сервер предназначен только для разработки и не подходит для использования в рабочей среде.
